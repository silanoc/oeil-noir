entite

génération des monstres et des personnages

class domaine.entite.Entite(**tous_les_monstres)

Une entité peut être un hèros ou un monstre. Dans tous les cas, ils ont des attributs et méthodes communes.

constructeur : * initialise via les valeurs d’un dictionnaire * attribue les pvencous en fonction de ceux max

est_vivant() → bool

regarde la valeur de pvencours. Vivant (donc true) si au mpoins 1 pv

:returns True ou False :rtype bool

jet_attaque(jet: int = 13) → bool

Defini si l’attaque est réussi, avec un 1d20 en paramètre (la fonction ne fait donc que comparer). Réussi si le jet est inférieur à la valeur d’attaque

Paramètres

jet (int) – valeur de jet de dé exécuté en paramètre

:returns reussite_attaque :rtype bool

jet_parade(jet: int = 6) → bool

Fait un test de parade avec 1d20 passé en paramètre. Fonction non refactorisé en 1 ligne, comme jet_attaque, au cas envie de rendre le code bavard. Réussi si le jet est inférieur à la valeur de parade

:returns reussite_parade :rtype bool

domaine.entite.generer_des_monstre() → list

Mettre dans une liste une instance d’Entité pour chaque monstre issus d’un dictionnaire. Ce dictionnaire provient d’un fichier json, lu par le port_json

Renvoie

liste_monstre

Type renvoyé

list
